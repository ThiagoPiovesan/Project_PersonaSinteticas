# Problem Description - Galaxies

---

## Technologies

- Python (for local execution uses UV package)
- Google Colab Notebook
- Streamlit

---

## Data

Location:

- data/dados_sinteticos.csv

Important notes:

- Informações demográficas (nome, data de nascimento, gênero, região)
- Respostas a surveys (múltipla escolha e categóricas com muitas
categorias)
- Histórico de compras (valor médio, frequência, canal preferido)

Care points:

- Empty fields
- Gender (masculino, feminino, nao_informado, outro, ...)
- Date formats and types
- categorical fields with many categories
- Values like 0 or 0.0 in numerical fields

---

## Main Goal

To segment consumers through cluster analysis, aiming to extract practical insights for the business team.

--

## Requirements

- Multi-agentic system
- Each cluster is embodied by an agent
- Allowing stakeholders to interact with the segments and gain conversational behavioral insights
- Google Colab notebook to develop the ML part
- The multi-agentic system must be able to run locally or any other platform
- Every part including the chain of thought must be well documented, comment every step and function, describe all the logic.

---

## Tasks

### Part 1 - Machine Learning (Clustering)

1. Treatment on Categorical Data
2. Feature Engineering with the Data
3. Simple plots of the data
4. Dimensionality Reduction
5. Clustering:
    a. use at leat 2 clustering algorithms (KMeans, KMedoids, GMM, DBSCAN, etc)
    b. Justify the choice of the algorithm and the number of clusters
    c. Evaluate using internal metrics
    d. Justify the result
6. Summary of Frequent values per cluster
    a. For each cluster, identify the most frequent categorical variables
    b. Produce a summary table: cluster size, mean values (numeric ones), modal (categorical) values.
    c. Indicate significant differences between clusters.

### Part 2 - Agentic Layer

1. Building the Pesona Agents
    a. From the cluster generated on Part 1, build an agent that represents each cluster.
    b. Each agent should embody the profile of the cluster it represents and be able to answer questions in First-Person like a tipyc member of that segment.
    c. The prompt/persona construction should be procedurally derived from the clustering results.

2. Agentic System Architecture
    a. Free framework (LangGraph, CrewAI, LangChain, AutoGen, AgnoAI or custom implementation)
    b. Justify the choice of the framework and adopted architecture
    c. No need for complex architecture - Priorize for clear an funcional over super advanced features

3. Interface
    a. Build the interface using Streamlit or similar that allow the Stakeholders to:
        1. "Selecionar com qual(is) persona(s) deseja conversar, ou conversar
            com todas simultaneamente."
        2. "Fazer perguntas de negócio (ex: "Como você reagiria a um aumento
            de preço de 10%?", "O que te faria trocar de canal?")."
        3. "Visualizar as respostas de cada agente lado a lado para
            comparação entre segmentos."

4. Insights generated by the agents
    a. Demonstrate at least 3 business related questions answered by the agents
    b. Comment about the coherence and consistency of the answers related to the statistic profile of each cluster
    c. Reflect on the limitations and potential uses of the approach.
